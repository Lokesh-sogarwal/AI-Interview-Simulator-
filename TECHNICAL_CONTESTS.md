# Technical Coding Contests - Weekly & Biweekly

## Overview

The contest system now supports **technical coding contests** with problems generated by Hugging Face AI. Contests can be scheduled as:
- **Weekly** - Every Sunday (or any day)
- **Biweekly** - Every two weeks on the same day

Each contest automatically generates coding interview problems in various difficulty levels using Hugging Face models.

## Features

✅ **Automated Problem Generation** - Uses Hugging Face AI to generate coding problems
✅ **Weekly/Biweekly Scheduling** - Automatic contest scheduling
✅ **Technical Focus** - LeetCode-style coding problems
✅ **Multiple Difficulty Levels** - Easy, Medium, Hard
✅ **Admin Control** - Only admins can create and schedule contests
✅ **Participant Tracking** - Leaderboards with ratings

## Contest Structure

### New Contest Fields

```typescript
interface Contest {
  name: string;
  type: "coding";           // Contest type
  frequency: "weekly" | "biweekly";
  problemCount: number;     // Number of problems (default: 3)
  problems: Problem[];      // Generated problems
  startDate: Date;
  endDate: Date;
  duration: number;         // Hours (e.g., 2, 3, 4)
  isRecurring?: boolean;    // For recurring contests
  recurringId?: string;     // Group ID for recurring contests
}
```

### Problem Structure

```typescript
interface Problem {
  title: string;                    // Problem title
  description: string;              // Detailed description
  difficulty: "Easy" | "Medium" | "Hard";
  constraints: string;              // Input/output constraints
  examples: Array<{
    input: string;
    output: string;
    explanation?: string;
  }>;
  topics: string[];                 // Topics: Array, String, DP, etc.
}
```

## API Endpoints

### 1. Create Contest (Admin Only)

```bash
POST /api/contests
Authorization: Bearer <admin-token>

{
  "name": "Weekly Coding Challenge",
  "description": "Test your coding skills",
  "startDate": "2026-06-06T14:00:00Z",
  "endDate": "2026-06-06T16:00:00Z",
  "problemCount": 3,
  "frequency": "weekly",
  "maxParticipants": 500
}
```

**Response:**
```json
{
  "ok": true,
  "id": "507f1f77bcf86cd799439011",
  "problemCount": 3
}
```

### 2. Schedule Recurring Contests (Admin Only)

```bash
POST /api/contests/schedule
Authorization: Bearer <admin-token>

{
  "name": "Weekly Coding Challenge",
  "frequency": "weekly",              // or "biweekly"
  "dayOfWeek": 5,                    // 0=Sunday, 5=Friday, 6=Saturday
  "startTime": "14:00",              // Contest start time (24-hour format)
  "duration": 2,                     // Contest duration in hours
  "problemCount": 3,                 // Problems per contest
  "maxParticipants": 500,
  "startDate": "2026-06-01T00:00:00Z",
  "endDate": "2026-12-31T23:59:59Z"   // When to stop creating recurring contests
}
```

**Response:**
```json
{
  "ok": true,
  "message": "Created 27 contests",
  "frequency": "weekly",
  "contests": [
    {
      "id": "507f1f77bcf86cd799439011",
      "startDate": "2026-06-06T14:00:00Z",
      "endDate": "2026-06-06T16:00:00Z"
    },
    // ... more contests
  ]
}
```

## Example: Create Weekly Contest for Saturdays

```bash
curl -X POST http://localhost:3000/api/contests/schedule \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Weekend Coding Marathon",
    "frequency": "weekly",
    "dayOfWeek": 6,
    "startTime": "10:00",
    "duration": 3,
    "problemCount": 4,
    "maxParticipants": 1000,
    "startDate": "2026-06-01T00:00:00Z",
    "endDate": "2026-12-31T23:59:59Z"
  }'
```

This creates a contest every Saturday at 10:00 AM for 3 hours, with 4 coding problems.

## Problem Generation

### How It Works

1. **Admin creates/schedules contest** → API receives request
2. **Hugging Face model generates problems** → Problems are created based on prompt
3. **Problems stored in contest** → Database saves problems with contest
4. **Users see problems** → Contest details page shows all problems

### Supported Models

- **Default:** `Qwen/Qwen2.5-1.5B-Instruct` (fast, efficient)
- **Fallback:** Built-in LeetCode-style problems if generation fails

### Problem Topics

Generated problems cover:
- Arrays & Strings
- Hash Tables & Sets
- Linked Lists
- Stacks & Queues
- Trees & Graphs
- Dynamic Programming
- Sorting & Searching

## Scoring System

```
Score = (Problems Solved / Total Problems) × 100

Rating Calculation:
- Base Rating: 1200
- Score Bonus: Score × 10
- Rank Bonus: (100 - Rank Percentile) × 5
- Speed Bonus: -1 point per minute over median time

Final Rating = 1200 + Score Bonus + Rank Bonus + Speed Bonus
```

## Troubleshooting

**Q: Problems not generating, getting defaults**
A: Check HUGGINGFACE_API_KEY is set and model is accessible

**Q: Contest started but still shows "Upcoming"**
A: Status updates based on startDate comparison. Check server time.

**Q: Too many/few problems generated**
A: Specify `problemCount` in request body (default: 3, max: 10)

**Q: Can't schedule recurring contests**
A: Only admins can use `/api/contests/schedule` endpoint
